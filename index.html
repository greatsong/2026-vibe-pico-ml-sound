소리를 배우는 피코 — 머신러닝 첫걸음

마이크 + k-NN으로 ‘내가 가르친 소리’(휘파람·박수·말소리·문·발걸음…)를 알아맞히는 만능 소리 분류기.

🎯 이 장을 마치면
  • 마이크로 소리를 숫자(특징)로 읽는다
  • 콘솔에서 키를 눌러 ‘내가 고른 소리들’을 직접 라벨링한다
  • k-NN으로 새 소리가 무엇인지 예측한다
  • 정규화·k·확신도·다중클래스 같은 머신러닝 핵심 개념을 이해한다
  • ML이 잘 되는 소리와 헷갈리는 소리의 차이를 안다
💡 왜 배우나요?

음성비서·새 찾기 앱·Shazam은 소리를 어떻게 알아들을까요? 비밀은 소리를 숫자로 바꿔, 닮은 것과 비교하는 거예요. 지금까지 피코는 if 값 > 임계값으로만 판단했지만, 소리는 규칙 한 줄로 못 가릅니다. 그래서 오늘은 다르게 — 내가 보여준 소리 예시만 모으면 피코가 스스로 경계를 찾습니다. 이것이 지도학습이고, 가장 직관적인 k-NN으로 ‘내 손이 만든 미니 음성 AI’를 만듭니다.

시작하기 전에 — 준비물과 순서

완전 처음이어도 괜찮아요. 아래 준비물만 확인하고 코드 ①→②→③→④ 순서로 한 칸씩 따라오면 됩니다. 막히면 각 단계의 ‘예상 화면’과 맨 끝 ‘자주 하는 실수’를 보세요.

준비물
피코 2 WH + 그로브 쉴드, 마이크(아날로그 또는 INMP441), WS2813 LED(2장에서 쓰던 것), USB 케이블. 새로 살 건 마이크뿐이에요.
진행 순서
① 마이크 값 보기 → ② 특징 뽑기 → ③ 소리 모으기 → ④ 예측. 한 코드씩 ▶로 실행합니다.
걸리는 시간
데이터 모으기 2~3분 + 예측까지, 천천히 해도 넉넉해요.
💡 마이크는 둘 중 아무거나 — 처음이면 아날로그
처음이면 그로브 아날로그 마이크(A1 포트에 꽂기만)로 시작하길 권해요. 익숙해지면 INMP441(음질↑)로 바꾸면 됩니다. 연결법은 아래 ‘하드웨어 준비’ 섹션에 있어요.

핵심 개념 — 기계가 소리를 배운다는 것

어려운 수학은 없어요. 오늘의 전부는 딱 한 문장 — ‘새 소리와 가장 닮은 이웃에게 물어본다’입니다. 먼저 도구와 개념을 천천히 짚어요.

지도학습
정답(라벨)이 붙은 예시로 배워, 새 데이터의 정답을 맞히는 것. 우리는 소리마다 이름(휘파람·박수…)을 알려줍니다.
k-NN
새 소리와 가장 닮은(가까운) 이웃 k개의 다수결로 결정. k는 ‘몇 명에게 물어볼까’예요.
‘학습’의 정체
k-NN은 훈련이 따로 없어요. 예시를 통째로 기억해뒀다 비교할 뿐. 그래서 모델 = 우리가 모은 데이터입니다.
특징(feature)
소리에서 뽑은 숫자 3개 — 크기(RMS)=얼마나 큰가, 높낮이(ZCR)=얼마나 높은가, 들쭉날쭉(crest)=톡 치는 소리(박수·노크)인가 이어지는 소리(목소리)인가.
🔑 정규화 — 단위가 다르면 공평하지 않다
세 특징의 범위가 제각각이에요. 높낮이는 0~수천, 크기는 0~수만, 들쭉날쭉은 1~수십. 그냥 거리를 재면 숫자가 큰 크기에만 휘둘립니다. 그래서 세 특징을 각각 0~1로 다시 스케일해 공평하게 만들어요. 머신러닝에서 거의 항상 필요한 단계랍니다.
🔬 더 알아보기 — 영교차(ZCR)가 어떻게 ‘높낮이’가 되나요?
소리는 0(가운데)을 중심으로 위아래로 출렁이는 물결이에요. 높은 소리일수록 물결이 빨라서 0을 더 자주 지나갑니다. 그래서 ‘0을 몇 번 지났나(영교차수)’를 세면 음의 높낮이를 대신할 수 있어요. 휘파람은 영교차가 많고, 낮은 ‘우~’ 허밍은 적습니다. FFT 같은 어려운 수학 없이도 음높이를 잡는 똑똑한 꼼수예요.
ℹ️ k 고르기 & 확신도
k=1은 이웃 하나에 휘둘려 과민(과적합), 너무 큰 k는 경계가 흐려져요. 보통 홀수 5. 확신도는 다수 비율 — 이웃 5 중 4가 ‘박수’면 ‘박수, 확신 80%’이고, 이걸 LED 칸 수로 보여줍니다.
ℹ️ 2가지에서 여러 가지로 — 다중클래스
성공/실패 같은 2개만이 아니라, 휘파람·박수·말소리·노크처럼 여러 종류도 똑같이 됩니다. 이웃 k명의 다수결에서 가장 표를 많이 받은 이름이 답이에요. 소리 종류마다 LED 색을 다르게 켭니다.

하드웨어 준비 — 마이크 연결하기 (아주 자세히)

마이크는 두 가지 중 하나만 쓰면 됩니다. 완전 처음이면 경로 A(아날로그)를 강력 추천 — 점퍼 없이 케이블만 꽂으면 되거든요. (둘 다 꽂아도 괜찮아요.) 결과를 보여줄 LED(2장에서 쓰던 것)도 함께 표시했어요 — D16 포트에 그로브 케이블로 꽂으면 됩니다. 아래 인터랙티브 배선표에서 포트·핀을 하나씩 체크하며 따라오세요.

🟢 경로 A · 그로브 아날로그 마이크 — 케이블만 꽂으면 끝

  1. 쉴드 확인피코가 그로브 쉴드 위에 잘 꽂혀 있는지 봅니다.
  2. 케이블 한쪽그로브 4핀 케이블 한쪽을 마이크 모듈에 ‘딸각’.
  3. 케이블 다른쪽다른쪽을 쉴드의 A1 포트에 ‘딸각’. (A0은 가스센서가 쓰니 꼭 A1!)
  4. 전원피코를 USB로 컴퓨터에 연결.
  5. 끝!점퍼·납땜 없이 이게 전부예요.

🔵 경로 B · INMP441 — 음질 업그레이드 (점퍼 5선)

⚠️ 먼저! VDD는 반드시 3.3V — 5V 금지
INMP441은 3.3V 소자라 VDD를 5V에 꽂으면 망가집니다. 쉴드가 피코 40핀을 모두 헤더로 뽑아주니, 그 헤더에 점퍼선(암-수)을 꽂으면 돼요. (색은 헷갈리지 않게 정한 것일 뿐, 달라도 됩니다.)
  1. VDD → 3V3빨강 점퍼 — ⚠ 5V 아님!
  2. GND → GND검정 점퍼
  3. SCK → GP18노랑 점퍼
  4. WS → GP19파랑 점퍼 — 반드시 SCK 바로 다음 핀
  5. SD → GP20초록 점퍼
  6. L/R → GND회색 점퍼 — 왼쪽 채널
💡 배선이 헷갈리면
‘배선 위젯/그림’의 핀맵을 보세요. INMP441은 그로브 포트가 아니라 헤더 핀에 점퍼로 꽂는다는 점만 기억하면 됩니다.

이제 값을 읽어 ‘소리가 숫자가 되는’ 걸 봅니다

코드 ① · 마이크 값 읽기 (아날로그 A1)PYTHON
# 소리 인식 ① — 마이크 값 읽기 (Grove 아날로그 마이크)
# 마이크를 그로브 쉴드의 'A1' 포트에 톡 꽂고 실행하세요. (A0은 가스센서가 쓰니 A1!)
# 휘파람·박수·말소리를 내면 숫자가 출렁여요. Thonny의 '플로터'를 켜면 그래프로 보입니다.
from machine import ADC, Pin
import time, math

mic = ADC(Pin(27))                 # A1 그로브 포트 = GP27

def level(n=300):                  # 짧게 n개를 재서 '소리 크기'(RMS)를 구함
    b = [mic.read_u16() for _ in range(n)]
    m = sum(b) / n                 # 평균(=무음일 때의 가운데 값)을 빼고
    return math.sqrt(sum((v - m) ** 2 for v in b) / n)   # 출렁임의 크기

print("준비됐어요! 휘파람·박수·말소리를 내보세요. (Thonny 플로터 추천)")
while True:
    print(level())
    time.sleep(0.05)

따라하기 ① 소리를 잡아 특징 뽑기

소리가 커지면 한 덩어리(약 0.1초)로 잡아, 크기(RMS)와 높낮이(ZCR) 두 숫자를 뽑습니다. 이 둘이 다음 단계 예측의 입력이에요.

코드 ② · 소리 감지 + 특징 추출PYTHON
# 소리 인식 ② — 소리를 한 덩어리로 잡아 '특징' 세 개 뽑기
# 크기(RMS)=얼마나 큰가 · 높낮이(ZCR)=얼마나 높은가 · 들쭉날쭉(crest)=톡 치는 소리인가
from machine import ADC, Pin
from neopixel import NeoPixel
import time, math

mic = ADC(Pin(27))
WIN_MS = 200       # 한 번에 '0.2초'를 한 덩어리로 (녹음·예측과 동일한 길이!)
def grab():
    buf = []; t0 = time.ticks_ms()
    while time.ticks_diff(time.ticks_ms(), t0) < WIN_MS:
        buf.append(mic.read_u16())
    return buf

def features(buf):                 # 한 조각 → (크기, 높낮이, 들쭉날쭉)
    n = len(buf); m = sum(buf) / n
    ss = 0.0; zc = 0; peak = 0.0; prev = buf[0] - m
    for v in buf:
        d = v - m; ad = d if d >= 0 else -d
        ss += d * d
        if ad > peak: peak = ad          # 가장 큰 순간값
        if (d >= 0) != (prev >= 0): zc += 1
        prev = d
    rms = math.sqrt(ss / n)
    return rms, zc, peak / (rms + 1e-9)  # 들쭉날쭉 = 최댓값 ÷ 평균크기

def level(n=120):
    b = [mic.read_u16() for _ in range(n)]; m = sum(b) / n
    return math.sqrt(sum((v - m) ** 2 for v in b) / n)

TIMING = (280, 515, 515, 745); NUM = 10
np = NeoPixel(Pin(16), NUM, timing=TIMING)
def flash(c, t=0.2): np.fill(c); np.write(); time.sleep(t); np.fill((0, 0, 0)); np.write()

print("조용히 해주세요… 배경소음 측정 중")
base = max(level() for _ in range(20)); THRESH = base * 2   # 민감하게(작은 소리도)
print("준비! 소리를 내보세요. (임계값=%.0f)" % THRESH)
while True:
    if level(120) > THRESH:
        rms, zcr, crest = features(grab())
        print("소리 감지!  크기=%.0f  높낮이=%d  들쭉날쭉=%.2f" % (rms, zcr, crest))
        flash((0, 0, 40)); time.sleep(0.4)
    time.sleep(0.005)

휘파람을 불면 셸에 소리 감지! 크기=320 높낮이=181 들쭉날쭉=3.2 처럼 떠요. 박수·노크는 들쭉날쭉이 크고, 목소리·허밍은 작게 나옵니다. 소리마다 세 숫자가 달라지면 성공!

• 조금 헷갈리는 소리 → 각 10개 권장(여기서 정확도가 거의 평평해져요. 15~20개로 늘려도 +2~3%p뿐).
• 많이 헷갈리거나 종류가 5개 이상 → 20~30개+ 필요(또는 아래 FFT 심화로 특징 늘리기).
개수보다 ‘균형’이 더 중요해요 — 한 소리만 30개·다른 건 3개면 30개짜리 쪽으로 쏠립니다. 소리마다 비슷한 수로!

def grab():
    buf = []; t0 = time.ticks_ms()
    while time.ticks_diff(time.ticks_ms(), t0) < WIN_MS:
        buf.append(mic.read_u16())
    return buf
def features(buf):
    n = len(buf); m = sum(buf) / n; ss = 0.0; zc = 0; peak = 0.0; prev = buf[0] - m
    for v in buf:
        d = v - m; ad = d if d >= 0 else -d; ss += d * d
        if ad > peak: peak = ad
        if (d >= 0) != (prev >= 0): zc += 1
        prev = d
    r = math.sqrt(ss / n); return r, zc, peak / (r + 1e-9)
def level(n=120):

    return math.sqrt(sum((v - m) ** 2 for v in b) / n)

from machine import ADC, Pin
from neopixel import NeoPixel

mic = ADC(Pin(27))
WIN_MS = 200       # 한 번에 '0.2초'를 한 덩어리로 (녹음과 동일한 길이여야 특징이 맞아요!)

    while time.ticks_diff(time.ticks_ms(), t0) < WIN_MS:
        buf.append(mic.read_u16())
    return buf
def features(buf):
    n = len(buf); m = sum(buf) / n; ss = 0.0; zc = 0; peak = 0.0; prev = buf[0] - m
    for v in buf:

        if ad > peak: peak = ad
        if (d >= 0) != (prev >= 0): zc += 1
        prev = d
    r = math.sqrt(ss / n); return r, zc, peak / (r + 1e-9)
def level(n=120):

    return math.sqrt(sum((v - m) ** 2 for v in b) / n)

# ── 1) 학습 = 데이터 기억 (특징 3개) ──
rows = []
for ln in open("sounds.csv"):
    ln = ln.strip()
    if ln == "" or ln.startswith("rms"): continue
    r, z, c, nm = ln.split(","); rows.append((float(r), float(z), float(c), nm))
names = sorted(set(r[3] for r in rows)); cidx = {nm: i for i, nm in enumerate(names)}
print("예시 %d개, 소리 종류: %s" % (len(rows), names))

# ── 2) 정규화 (세 특징 각각 0~1) ──
mins = [min(r[i] for r in rows) for i in range(3)]
maxs = [max(r[i] for r in rows) for i in range(3)]
def norm(v): return [(v[i] - mins[i]) / (maxs[i] - mins[i] + 1e-9) for i in range(3)]
ex = [(norm(r[:3]), r[3]) for r in rows]

# ── 3) k-NN (거리 가중) ──
def predict(feat, k=5):
    q = norm(feat)
    def d2(e): return sum((e[0][i] - q[i]) ** 2 for i in range(3))
    near = sorted(ex, key=d2)[:k]
    vote = {}
    for e in near:
        vote[e[1]] = vote.get(e[1], 0) + 1.0 / (d2(e) + 1e-6)   # 가까울수록 큰 표
    best = max(vote, key=vote.get)
    return best, vote[best] / sum(vote.values())

# ── 4) 새 소리 기다렸다 예측 ──
print("조용히… 배경소음 측정 중")
base = max(level() for _ in range(20)); THRESH = base * 2   # 민감하게(작은 소리도 잡게). 잦으면 ↑
print("준비! 소리를 내면 알아맞힐게요.")
while True:
    if level(120) > THRESH:
        feat = features(grab()); name, conf = predict(feat)
        print("이건… %s!  (확신 %d%%, 크기=%.0f 높낮이=%d 들쭉=%.2f)"
              % (name, conf * 100, feat[0], feat[1], feat[2]))
        show(cidx[name], max(1, int(conf * NUM))); time.sleep(1.0); show(0, 0)
    time.sleep(0.005)

# 소리 인식 ①(I2S) — INMP441 마이크 값 읽기 (음질 업그레이드 버전)
# 점퍼 연결: VDD→3.3V(⚠5V 금지), GND→GND, SCK→GP18, WS→GP19, SD→GP20, L/R→GND
from machine import I2S, Pin
import struct, math, time

audio = I2S(0, sck=Pin(18), ws=Pin(19), sd=Pin(20),
            mode=I2S.RX, bits=32, format=I2S.MONO, rate=16000, ibuf=40000)
buf = bytearray(512 * 4)

def level():                       # 짧게 512표본의 '소리 크기'(RMS)
    audio.readinto(buf)
    v = [x >> 16 for x in struct.unpack("<512i", buf)]   # 32→16비트로 줄여 보기 쉽게
    m = sum(v) / 512
    return math.sqrt(sum((x - m) ** 2 for x in v) / 512)

print("준비됐어요! 휘파람·박수·말소리를 내보세요. (Thonny 플로터 추천)")
while True:
    print(level())
    time.sleep(0.05)

INMP441 심화 — FFT 주파수밴드로 음색까지 (성능 끌어올리기)

예측이 잘 돼야 재밌죠. 더 어려운 소리(휘파람 vs 쉬익, 모음 ‘아’ vs ‘이’처럼 음색만 다른 소리)까지 가르려면 특징을 더 풍부하게 — 8개 주파수 대역의 에너지를 씁니다. 음성비서가 쓰는 방식의 축소판이에요. (INMP441 권장, ulab 없이 Goertzel로 계산.)

# 소리 인식 ⑤ (심화 · INMP441) — FFT 주파수밴드 특징으로 더 똑똑하게
# 8개 주파수 대역의 에너지를 특징으로 쓰면 '음색(timbre)'까지 구분해요.
# ulab 없이 Goertzel 알고리즘으로 각 대역 에너지를 직접 계산합니다.
# 한 번 실행에서 [수집] → 'go' 입력 → [예측] 흐름으로 끝납니다.
from machine import I2S, Pin
from neopixel import NeoPixel
import struct, time, math

audio = I2S(0, sck=Pin(18), ws=Pin(19), sd=Pin(20),
            mode=I2S.RX, bits=32, format=I2S.MONO, rate=16000, ibuf=40000)
N = 1600; raw = bytearray(N * 4)
def grab():
    audio.readinto(raw)
    return [x >> 16 for x in struct.unpack("<%di" % N, raw)]

BANDS = [200, 350, 600, 1000, 1700, 2800, 4500, 7000]   # 8개 대역 중심 주파수(Hz)
def goertzel(x, f):                 # 특정 주파수 f의 에너지(파워)
    k = 2 * math.cos(2 * math.pi * f / 16000); s1 = 0.0; s2 = 0.0
    for v in x:
        s0 = v + k * s1 - s2; s2 = s1; s1 = s0
    return s1 * s1 + s2 * s2 - k * s1 * s2
def features(x):                    # 음량과 무관한 '스펙트럼 모양' 8개
    m = sum(x) / len(x); x = [v - m for v in x]
    e = [max(goertzel(x, f), 0.0) for f in BANDS]; tot = sum(e) + 1e-9
    return [math.log(ei / tot + 1e-6) for ei in e]

small = bytearray(512 * 4)
def level():
    audio.readinto(small); v = [x >> 16 for x in struct.unpack("<512i", small)]; m = sum(v) / 512
    return math.sqrt(sum((x - m) ** 2 for x in v) / 512)

TIMING = (280, 515, 515, 745); NUM = 10
np = NeoPixel(Pin(16), NUM, timing=TIMING)
PALETTE = [(40, 0, 0), (0, 40, 0), (0, 0, 40), (40, 40, 0), (40, 0, 40), (0, 40, 40)]
def show(ci, lv):
    c = PALETTE[ci % len(PALETTE)]
    for i in range(NUM): np[i] = c if i < lv else (0, 0, 0)
    np.write()

# ★ 음색이 다른 소리일수록 효과 큼 (예: 휘파람 vs 쉬익, 모음 '아' vs '이') ★
LABELS = {"1": "휘파람", "2": "말소리", "3": "노크", "4": "박수"}
D = len(BANDS)
print("조용히… 배경소음 측정 중")
base = max(level() for _ in range(20)); THRESH = base * 3 + 30

data = []
print("\n[수집] 숫자키 누르고 소리내기 · go=예측 시작")
for k, v in LABELS.items(): print("  %s = %s" % (k, v))
while True:
    cmd = input("라벨(go=예측)> ").strip()
    if cmd == "go": break
    nm = LABELS.get(cmd)
    if not nm: continue
    print("  ▶ '%s' 소리를 지금 내세요… (3초)" % nm)
    best = None; bestr = -1.0; t0 = time.ticks_ms()
    while time.ticks_diff(time.ticks_ms(), t0) < 3000:        # 3초 중 '가장 또렷한' 조각
        x = grab(); mm = sum(x) / len(x); rr = sum((v - mm) ** 2 for v in x)
        if rr > bestr: bestr = rr; best = features(x)
    data.append((best, nm)); print("  저장 (총 %d개)" % len(data))

names = sorted(set(d[1] for d in data)); cidx = {n: i for i, n in enumerate(names)}
mins = [min(d[0][i] for d in data) for i in range(D)]
maxs = [max(d[0][i] for d in data) for i in range(D)]
def norm(f): return [(f[i] - mins[i]) / (maxs[i] - mins[i] + 1e-9) for i in range(D)]
ex = [(norm(d[0]), d[1]) for d in data]
def predict(f, k=5):
    q = norm(f)
    def d2(e): return sum((e[0][i] - q[i]) ** 2 for i in range(D))
    near = sorted(ex, key=d2)[:k]; vote = {}
    for e in near: vote[e[1]] = vote.get(e[1], 0) + 1.0 / (d2(e) + 1e-6)
    b = max(vote, key=vote.get); return b, vote[b] / sum(vote.values())

print("\n[예측] 소리를 내보세요.")
while True:
    if level() > THRESH:
        nm, conf = predict(features(grab()))
        print("이건… %s!  (확신 %d%%)" % (nm, conf * 100))
        show(cidx[nm], max(1, int(conf * NUM))); time.sleep(1.0); show(0, 0)

특징 2개(0.73) → 3개(0.96) → FFT 8밴드는 음색이 비슷한 소리까지 잡아냅니다. 단 계산이 무거워 INMP441(깨끗한 16kHz)에서 제일 잘 돼요. ‘성능 = 좋은 특징’임을 몸으로 확인하는 코너.

피코가 말을 한다 — 소리의 주인공을 목소리로

k-NN이 알아맞힌 소리에게 ‘이름’을 붙여 목소리로 말해주는 출력기. Grove MP3 v4로 듣기→생각→말하기 한 사이클을 완성합니다.

🎯 이 장을 마치면
  • 출력 모듈(Grove MP3 v4)에 UART로 ‘명령’을 보내 소리를 낸다
  • 분류 결과(라벨)를 음성 트랙 번호로 잇는 ‘룩업 테이블’을 이해한다
  • ‘자기 소리 되먹임’과 ‘모듈 부팅’ 같은 현실 문제를 코드로 다스린다
  • 확신이 낮으면 ‘모르겠어요’ 하고 침묵하는 규칙을 넣는다
  • 입력(마이크)→처리(k-NN)→출력(음성)의 AI 한 바퀴를 직접 잇는다
💡 왜 배우나요?

소리 분류 챕터에서 피코는 ‘이건 휘파람!’이라고 속으로만 알았어요(콘솔 글자·LED 색). 그런데 음성비서·새 찾기 앱은 거기서 한 발 더 나아가 사람의 말로 대답합니다. 오늘은 그 마지막 한 칸을 채워요. 분류 결과라는 추상적인 라벨을, Grove MP3 v4를 통해 들리는 목소리로 되돌려줍니다. 그러면 ‘기계가 듣고·생각하고·말한다’는 AI의 한 사이클이 눈앞에서 완성돼요. 아래 순서대로 한 칸씩만 따라오면 됩니다.

0단계 · 시작하기 전에 (준비물 점검)

이 챕터는 소리 분류 챕터의 ‘다음 편’이에요. 시작 전에 아래 세 가지가 준비됐는지 먼저 확인하세요. 하나라도 비면 그 챕터를 먼저 끝내고 오면 됩니다.

  • 소리 분류 챕터에서 만든 sounds.csv가 피코 안에 들어 있다 (이게 ‘모델’이에요)
  • 마이크가 연결돼 있고, 휘파람·박수가 이미 잘 구분됐다
  • 새로 준비한 Grove MP3 v4 모듈과 작은 스피커(또는 이어폰)가 손에 있다
ℹ️ 완성하면 이런 모습이에요 (미리 그려보기)
마이크에 대고 휘파람을 붑니다 → 피코가 0.5초 생각하더니 → 스피커에서 “휘파람이에요!” 하고 말합니다. 박수를 치면 “박수네요!”. 애매한 소리엔 조용히 있어요. 이 한 장면을 만드는 게 오늘의 목표예요.
🔑 오늘 추가되는 건 ‘입’ 하나뿐
듣기(마이크)·생각하기(k-NN)는 이미 다 만들었어요. 오늘은 거기에 말하는 입(MP3 모듈)만 답니다. 그래서 새로 배우는 코드는 생각보다 적어요.

핵심 개념 — ‘출력기’가 마지막 한 칸을 채운다

코드를 짜기 전에, 오늘 쓰는 부품과 개념을 천천히 짚어요. 딱 네 가지만 이해하면 됩니다.

입력 vs 출력
마이크는 소리를 듣는 입력, Grove MP3 v4는 소리를 내는 출력이에요. 방향이 정반대. 둘을 이으면 ‘듣고 대답하는 기계’가 됩니다.
MP3 v4 = ‘똑똑한 부하 직원’
피코는 “3번 곡 틀어”라는 명령만 보내요. 파일 찾기·소리 키우기·스피커 울리기는 모듈이 알아서 해요. 피코는 소리를 직접 안 만듭니다.
UART(시리얼)
두 기계가 전선 한 줄로 글자를 주고받는 약속이에요. 마치 두 사람이 실 전화기로 속닥이는 것. 피코의 TX(말하는 핀)와 모듈의 RX(듣는 핀)를 잇습니다.
룩업 테이블
라벨 → 트랙 번호 표예요. ‘휘파람’이면 1번, ‘박수’면 2번… 분류 결과를 음성에 잇는 짧은 사전 하나가 오늘의 핵심 코드입니다.
🔑 오늘의 한 문장 — ‘라벨을 목소리로 바꾸는 사전’
k-NN이 내놓는 답은 "휘파람" 같은 글자(라벨)예요. 이걸 TRACK = {"휘파람":1, "박수":2}라는 사전 한 줄로 음성 트랙 번호에 연결합니다. 그게 전부예요.
ℹ️ 확신이 낮으면 — 침묵이 정답
분류가 흔들릴 때(이웃이 갈릴 때) 자신 있게 틀린 답을 말하면 더 나빠요. 그래서 확신도가 기준(60%) 아래면 아무 말도 안 합니다. ‘모를 땐 모른다고 하는 것’ — 좋은 AI의 태도이고, 한 줄(if conf >= CONF_MIN)로 가르칠 수 있어요.

1단계 · 모듈 연결하기 (배선)

Grove MP3 v4는 그로브 4핀 케이블 하나면 연결 끝이에요. 마이크는 앞 챕터 그대로 두고, MP3 모듈만 새로 답니다.

  1. MP3 모듈 꽂기그로브 케이블로 쉴드의 UART 포트에 ‘톡’ 꽂기 (피코 핀 TX=GP0, RX=GP1에 연결돼요)
  2. 스피커 달기모듈의 스피커 단자에 작은 스피커, 또는 3.5mm 잭에 이어폰을 꽂기
  3. SD카드 꽂기음성 MP3를 넣은 microSD를 모듈에 삽입 (만드는 법은 2단계)
⚠️ 점퍼로 직접 연결한다면 — TX와 RX는 ‘엇갈려’ 꽂아요
그로브 포트로 꽂으면 보통 알아서 맞아요. 하지만 점퍼선으로 직접 연결한다면, 내가 말하는 선(TX)은 상대가 듣는 선(RX)에 닿아야 합니다.
• 피코 TX(GP0) → 모듈 RX
• 피코 RX(GP1) → 모듈 TX
• 모듈 VCC → 5V, GND → GND
소리가 안 나면 이 두 신호선을 서로 바꿔 끼우는 게 1순위 점검이에요.
  • MP3 모듈의 전원 LED에 불이 들어오나요?
  • 스피커(또는 이어폰)가 모듈에 단단히 꽂혔나요?
  • SD카드가 딸깍 소리 나게 끝까지 들어갔나요?

2단계 · 음성 파일 만들어 SD카드에 넣기

소리 이름마다 한 마디 음성을 만들어 0001.mp3, 0002.mp3… 순서대로 SD카드에 넣습니다. 여기서 파일 이름의 숫자 = 트랙 번호가 돼요.

ℹ️ 샘플 음성 바로 받기 — 직접 안 만들어도 돼요
아래 4개를 받아 FAT32 카드 루트에 0001→0004 순서대로 하나씩 넣으면 끝이에요(멘트: 휘파람·박수·말소리·노크). 직접 만들고 싶으면 아래 단계대로 TTS로 만들어도 됩니다.
⬇ 0001 휘파람⬇ 0002 박수⬇ 0003 말소리⬇ 0004 노크⬇ 전체 ZIP
ℹ️ 폰 마이크로 직접 녹음해 만들기 (MP3/WAV 다운로드)
TTS 대신 내 목소리로 만들고 싶으면 아래 도구로 녹음하세요. 버튼 누르고 2초간 또박또박 → 미리듣기 → MP3 또는 WAV로 다운로드 → SD에 넣기. (https 사이트에서 마이크 권한 허용 필요. WAV도 v4.0이 재생합니다.)
  1. 멘트 정하기라벨마다 한 마디. 예) 1번=“휘파람이에요” / 2번=“박수네요” / 3번=“말소리예요” / 4번=“노크 소리예요”
  2. 음성 만들기직접 녹음하거나, TTS(읽어주기) 사이트·앱으로 mp3를 생성
  3. 이름 붙이기0001.mp3=1번 라벨, 0002.mp3=2번… 룩업 테이블 순서대로 정확히
  4. SD카드에 복사SD카드 맨 위 폴더(루트)에 0001부터 순서대로 하나씩 복사 → 모듈에 삽입
🔑 SD카드는 ‘FAT32’로 포맷 — NTFS·exFAT는 안 돼요
이 모듈은 FAT32(또는 FAT16)만 읽어요. NTFS·exFAT로 포맷하면 인식이 안 됩니다. 카드는 32GB 이하를 권장해요(더 큰 카드는 FAT32 포맷이 까다로워요).
• Windows: 카드 우클릭 → 포맷 → 파일 시스템 FAT32 → 시작. (FAT32가 안 보이면 32GB 이하 카드를 쓰거나 ‘guiformat’ 같은 도구 사용)
• Mac: 디스크 유틸리티 → 카드 선택 → 지우기 → 포맷 MS-DOS(FAT) → 지우기.
⚠️ Mac이라면 — ‘숨김 파일’이 트랙 번호를 밀어요
맥에서 카드에 복사하면 ._0001.mp3·.Spotlight-V100·.Trashes 같은 숨김 파일이 함께 생겨요. 이게 트랙 번호를 한 칸씩 밀어버려 엉뚱한 곡이 나옵니다. → 복사 후 터미널에서 dot_clean /Volumes/카드이름을 실행하고 휴지통을 비우거나, Windows PC에서 복사하는 게 가장 깔끔해요.
⚠️ ★ 꼭! — ‘번호 순서대로 하나씩’ 복사하기
이 모듈(WT2605CX)은 AT+PLAY=sd0,N의 N을, 파일 이름이 아니라 복사된 순서로 매기는 경우가 많아요. 그래서 여러 파일을 한꺼번에 끌어다 놓으면 순서가 뒤죽박죽이 될 수 있습니다. 안전하게 하려면 SD카드를 FAT32로 포맷한 뒤, 0001.mp3 → 0002.mp3 … 순서로 한 개씩 복사하세요. (엉뚱한 곡이 나오면 거의 이 문제예요.)
🤖AI에게 TTS 음성 만들기 시키기 (그대로 복사)
휘파람·박수·말소리·노크, 이 네 가지에 대해 한국어로 한 마디씩 멘트(예: '휘파람이에요', '박수네요', '말소리예요', '노크 소리예요')를 읽어주는 mp3 파일을 만들고 싶어. 무료 TTS로 만드는 구체적인 방법(사이트나 앱 추천 포함)을 알려주고, 파일 이름을 0001.mp3~0004.mp3로 붙이는 순서까지 정리해줘. 음성은 너무 길지 않게 1~2초로.
💡 음성 대신 효과음도 좋아요
꼭 말일 필요는 없어요. “딩동”·“삐빅”·동물 울음 같은 효과음을 넣어도 재밌습니다. 중요한 건 라벨마다 다른 트랙이라는 점뿐이에요.

3단계 · 모듈만 따로 테스트 (소리부터 확인!)

분류와 합치기 전에, MP3 모듈이 혼자서도 소리를 잘 내는지부터 확인해요. 여기서 소리가 나야 다음 단계가 의미 있어요. 제일 먼저 넘어야 할 관문이에요.

코드 ① · MP3 모듈 단독 테스트 (1·2번 곡 재생)PYTHON
# MP3 출력 ① — Grove MP3 v4.0 (WT2605CX) 모듈 단독 테스트
# 피코는 "몇 번 곡 틀어/볼륨 얼마로" 같은 '명령'만 보냅니다. 재생은 모듈이 해요.
# ⚠ v2.0(KT403A)과 명령이 '완전히' 다릅니다! v4.0은 7E..EF 바이너리가 아니라
#    'AT+' 텍스트 명령을 \r\n 으로 끝맺어 115200 baud로 보냅니다.
from machine import UART, Pin
import time

# Grove 케이블 → 쉴드의 UART 포트. 피코 TX=GP0, RX=GP1
uart = UART(0, baudrate=115200, tx=Pin(0), rx=Pin(1))   # ★ 115200! (v2.0은 9600)
time.sleep(1.0)                                          # 모듈 부팅 대기

def at(cmd, wait=0.4):
    uart.write((cmd + "\r\n").encode())                  # AT 텍스트 + 줄바꿈
    print(">", cmd)
    time.sleep(wait)
    if uart.any():
        print("  <", uart.read())                        # 모듈 응답(보통 'OK')

at("AT+VOL=22")                                          # 볼륨 0~31 (시끄러우면 낮추기)
print("1번 곡(0001.mp3) 재생…")
at("AT+PLAY=sd0,1")                                      # SD카드 루트의 1번 곡
time.sleep(3)
print("2번 곡(0002.mp3) 재생…")
at("AT+PLAY=sd0,2")
time.sleep(3)
at("AT+STOP")
print("끝! 소리가 났나요?")

💡 여기서 잠깐 🤔 — 코드가 하는 일
Grove MP3 v4.0(WT2605CX)은 ‘AT+’ 텍스트 명령을 115200 baud로 받아요(v2.0의 7E…EF 바이너리와 완전 다름!). 맨 위에서 연결하고 모듈 부팅을 1초 기다린 뒤, AT+VOL=22로 볼륨(0~31)을, AT+PLAY=sd0,1로 1번 곡을 재생합니다. 명령 끝엔 항상 \r\n(줄바꿈)이 붙어요.
⚠️ 소리가 안 나면 — 이 순서로 점검
① 보드레이트 115200 맞나?(v4.0 필수) ② TX↔RX가 엇갈려 꽂혔나?(1순위) ③ 볼륨이 너무 작나?(AT+VOL=22→28) ④ SD가 FAT32이고 0001.mp3가 있나? ⑤ 부팅 대기(1.0→2.0초) 늘리기 ⑥ 응답(< OK)이 오는지. 대부분 ①·②·④에서 해결돼요.
  • 스피커에서 1번 곡 소리가 나나요?
  • 약 3초 뒤 2번 곡으로 바뀌나요?
  • AT+VOL=22의 숫자를 5·31로 바꾸면 음량이 변하나요?

4단계 · 분류 결과를 ‘목소리’로 (메인 완성)

이제 마지막! 마이크로 들은 소리를 k-NN으로 알아맞히고, 그 라벨에 맞는 트랙을 재생합니다. 코드의 TRACK 사전만 내 라벨에 맞추면 끝이에요.

코드 ② · k-NN 예측 → 음성 출력 (+LED)PYTHON
# MP3 출력 ② — k-NN이 알아맞힌 소리의 '주인공'을 목소리로 (Grove MP3 v4.0 · WT2605CX)
# 듣기(마이크) → 생각(k-NN) → 말하기(MP3).  ⚠ v4.0은 AT 명령 / 115200 baud!
# ※ 소리 분류 챕터에서 만든 sounds.csv(rms,zcr,crest,label)가 피코에 들어 있어야 해요.
import time, math
from machine import ADC, Pin, UART
from neopixel import NeoPixel

# ── 마이크 (아날로그 A1) — 소리 챕터와 똑같이 '듣기' · 특징 3개 ──
mic = ADC(Pin(27))
WIN_MS = 200       # 한 번에 '0.2초'를 한 덩어리로 (소리 챕터와 동일한 길이!)
def grab():
    buf = []; t0 = time.ticks_ms()
    while time.ticks_diff(time.ticks_ms(), t0) < WIN_MS:
        buf.append(mic.read_u16())
    return buf
def features(buf):
    n = len(buf); m = sum(buf) / n; ss = 0.0; zc = 0; peak = 0.0; prev = buf[0] - m
    for v in buf:
        d = v - m; ad = d if d >= 0 else -d; ss += d * d
        if ad > peak: peak = ad
        if (d >= 0) != (prev >= 0): zc += 1
        prev = d
    r = math.sqrt(ss / n); return r, zc, peak / (r + 1e-9)
def level(n=120):
    b = [mic.read_u16() for _ in range(n)]; m = sum(b) / n
    return math.sqrt(sum((v - m) ** 2 for v in b) / n)

# ── MP3 출력기 (Grove MP3 v4.0 · WT2605CX) — '말하기' : AT 명령 ──
uart = UART(0, baudrate=115200, tx=Pin(0), rx=Pin(1))   # ★ v4.0 = 115200
time.sleep(1.0)                                          # 모듈 부팅 대기
def at(s):
    uart.write((s + "\r\n").encode()); time.sleep(0.1)
at("AT+VOL=22")                                          # 볼륨 0~31

# ── ★ 라벨 → 트랙 번호 (룩업 테이블) ──
# SD(FAT32)에 0001.mp3="휘파람이에요", 0002.mp3="박수네요" … 순서대로 넣어두세요.
TRACK = {"휘파람": 1, "박수": 2, "말소리": 3, "노크": 4}

# ── LED (소리마다 색) ──
TIMING = (280, 515, 515, 745); NUM = 10
np = NeoPixel(Pin(16), NUM, timing=TIMING)
PALETTE = [(40, 0, 0), (0, 40, 0), (0, 0, 40), (40, 40, 0), (40, 0, 40), (0, 40, 40)]
def paint(c, lv):
    for i in range(NUM): np[i] = c if i < lv else (0, 0, 0)
    np.write()

# ── sounds.csv 불러오기 + 정규화 + 거리가중 k-NN (소리 챕터와 동일 · 특징 3개) ──
rows = []
for ln in open("sounds.csv"):
    ln = ln.strip()
    if ln == "" or ln.startswith("rms"): continue
    r, z, c, nm = ln.split(","); rows.append((float(r), float(z), float(c), nm))
names = sorted(set(r[3] for r in rows)); cidx = {nm: i for i, nm in enumerate(names)}
mins = [min(r[i] for r in rows) for i in range(3)]
maxs = [max(r[i] for r in rows) for i in range(3)]
def norm(v): return [(v[i] - mins[i]) / (maxs[i] - mins[i] + 1e-9) for i in range(3)]
ex = [(norm(r[:3]), r[3]) for r in rows]
def predict(feat, k=5):
    q = norm(feat)
    def d2(e): return sum((e[0][i] - q[i]) ** 2 for i in range(3))
    near = sorted(ex, key=d2)[:k]; vote = {}
    for e in near: vote[e[1]] = vote.get(e[1], 0) + 1.0 / (d2(e) + 1e-6)
    b = max(vote, key=vote.get); return b, vote[b] / sum(vote.values())

# ── 배경소음 → 임계값 ──
print("조용히… 배경소음 측정 중")
base = max(level() for _ in range(20)); THRESH = base * 2   # 민감하게(작은 소리도)

# ── ★ '말하기' — 자기 소리 되먹임 방지(말하는 동안 + 잔향까지 귀 닫기) ──
SPEAK_SEC = 2.5      # 멘트 길이만큼. 음성이 잘리면 늘리세요.
def say(track):
    at("AT+PLAY=sd0,%d" % track)         # ← 그 소리의 '이름'을 목소리로!
    time.sleep(SPEAK_SEC)
    t0 = time.ticks_ms()
    while level(120) > THRESH and time.ticks_diff(time.ticks_ms(), t0) < 2000:
        time.sleep(0.05)

CONF_MIN = 0.6       # 확신이 이보다 낮으면 '모르겠어요' → 침묵
print("준비! 소리를 내면 누구인지 말해줄게요. (멈추려면 Ctrl+C)")
while True:
    if level(120) > THRESH:
        feat = features(grab()); name, conf = predict(feat)
        if conf >= CONF_MIN and name in TRACK:
            print("이건… %s!  (확신 %d%%)" % (name, conf * 100))
            paint(PALETTE[cidx[name] % len(PALETTE)], max(1, int(conf * NUM)))
            say(TRACK[name])
        else:
            print("음… 잘 모르겠어요 (확신 %d%%) — 조용히 있을게요" % (conf * 100))
            paint((8, 8, 8), NUM); time.sleep(0.8)
        paint((0, 0, 0), 0)
    time.sleep(0.005)

⚠️ 여기가 핵심! 🔁 — ‘말하는 동안 귀를 닫는’ 이유
스피커가 “휘파람이에요”라고 말하면 그 소리를 마이크가 다시 듣고 또 판단해버려요 → 끝없이 자기 말을 따라 하는 무한 반복! 그래서 say()는 ① 말하는 동안 + ② 잔향이 빠질 때까지 입력을 잠시 무시합니다(SPEAK_SEC). 음성이 길어 끝까지 안 나오면 이 값을 늘리세요. 이게 이 챕터가 실제로 돌아가게 하는 장치예요.

TRACK = {"휘파람":1, ...}의 왼쪽 이름은 데이터를 모을 때 쓴 라벨과 한 글자도 다르면 안 됩니다. ‘휘파람’과 ‘휘파람소리’는 다른 이름이라 음성이 안 나와요. 라벨을 바꿨다면 여기도 같이 고치세요.

원인TX·RX가 같은 방향으로 꽂힘(안 엇갈림).
해결피코 TX↔모듈 RX로 두 신호선을 바꿔 끼우세요.
❌ 첫 곡만 안 나옴
원인모듈 부팅 전에 명령이 감.
해결맨 위 부팅 대기(1초)를 2초로 늘려 보세요.
❌ 엉뚱한 곡이 나옴
원인파일 복사 순서와 TRACK 값이 어긋남.
해결SD카드를 포맷하고 0001.mp3부터 순서대로 다시 복사.
❌ 특정 라벨만 침묵
원인TRACK 이름이 라벨과 글자가 다름.
해결sounds.csv 라벨과 똑같이 적으세요.
❌ 자기 말을 무한 반복
원인스피커 소리를 마이크가 또 들음.
해결SPEAK_SEC를 음성 길이만큼 넉넉히, 또는 스피커를 마이크에서 떼기.
❌ 맞혔는데 말을 안 함
원인확신이 CONF_MIN보다 낮음.
해결정상이에요. 데이터를 더 모으거나 기준값을 낮춰보세요.
❌ 소리가 너무 크거나 작음
원인볼륨 설정값.
해결AT+VOL=22의 숫자(0~31)를 조절하세요.

스스로 점검하기

다음 질문에 답할 수 있으면 이 장을 이해한 거예요. (먼저 스스로 답해보고 펼치세요)

마이크와 MP3 모듈은 뭐가 다른가요?
마이크는 소리를 듣는 입력, MP3 v4는 소리를 내는 출력. 방향이 반대예요.
피코가 MP3 모듈에 보내는 건 무엇인가요?
소리가 아니라 “몇 번 곡 틀어”라는 명령(UART). 재생은 모듈이 해요.
룩업 테이블(TRACK)은 무슨 일을 하나요?
분류 결과인 라벨을 트랙 번호로 이어줍니다. 오늘의 핵심 한 줄.
말하는 동안 마이크 입력을 왜 무시하나요?
스피커 소리를 마이크가 다시 듣고 무한 반복하는 ‘되먹임’을 막기 위해서예요.
확신이 낮으면 왜 말을 안 하나요?
자신 있게 틀린 답을 하는 것보다 ‘모른다’가 낫기 때문. 좋은 AI의 태도예요.
소리가 아예 안 날 때 1순위 점검은?
TX↔RX가 엇갈려 꽂혔는지, 그리고 SD에 0001.mp3가 있는지.
